Section 3 · CodeCommit

CodeCommit — Your AWS-Native Git

  • ▸Fully managed Git repository — no servers to run, no GitHub billing surprises
  • ▸IAM-authenticated — HTTPS with Git credentials OR SSH with IAM key pair. No OAuth tokens.
  • ▸Encrypted at rest and in transit — automatically, no config needed
  • ▸CloudTrail integration — every push, pull, clone is logged. Audit-ready.
  • ▸Triggers and notifications — push events can trigger Lambda, SNS, CodePipeline
  • ▸Branch protection — approval rule templates to enforce PR reviews before merge

Section 3 · CodeBuild

CodeBuild — Managed Build Service

  • ▸Fully managed — no build servers to provision, patch, or scale
  • ▸Runs your buildspec.yml — 4 phases: install → pre_build → build → post_build
  • ▸Managed environments — Amazon Linux 2, Ubuntu, Windows; multiple runtimes (Node, Python, Java, Go...)
  • ▸Pay per build minute — no idle costs. First 100 build-minutes/month free.
  • ▸CloudWatch Logs — every build log line is stored and searchable
  • ▸Environment variables — inject secrets from SSM Parameter Store or Secrets Manager at build time
  • ▸VPC support — build inside your VPC to access private resources
📦 What CodeBuild Produces

A deployment artifact — a .zip file containing your app files, appspec.yml, and scripts. This artifact is uploaded to S3. CodeDeploy downloads it from there.

⚡ Build Environments (Standard)
  • ▸aws/codebuild/standard:7.0 — Node 18, Python 3.11, Java 17, Go 1.21
  • ▸aws/codebuild/amazonlinux2-x86_64-standard:5.0 — AL2 with AWS CLI pre-installed
💰 Pricing

build.general1.small = $0.005/min. A typical 2-minute build costs $0.01. At 50 deploys/day that's $0.50/day. Basically free.

Section 3 · buildspec.yml

buildspec.yml — Tell CodeBuild What To Do

version: 0.2

env:
  parameter-store:
    DB_URL: /myapp/prod/db-url   # pull from SSM

phases:
  install:
    runtime-versions:
      nodejs: 18
    commands:
      - npm install

  pre_build:
    commands:
      - npm run lint            # fail here = nothing ships
      - npm test
      - npm audit --audit-level=high  # security scan

  build:
    commands:
      - echo "Build started `date`"
      - npm run build

  post_build:
    commands:
      - echo "Build complete `date`"

artifacts:
  files:
    - app/**/*        # your app files
    - appspec.yml     # must be in artifact
    - scripts/**/*    # lifecycle scripts
  discard-paths: no
  • 1install: Runtime + package install. Runs once. Cached between builds for speed.
  • 2pre_build: Lint, test, security scan. This is your quality gate. Fail here = pipeline stops.
  • 3build: The actual compile / bundle step. Produces deployable output.
  • 4post_build: Cleanup, notifications, tagging Docker images, etc.
  • 5artifacts: What gets zipped and uploaded to S3. CodeDeploy will download this exact package.
⚠ Common Mistake

Forgetting to include appspec.yml and scripts/ in the artifacts block. CodeDeploy needs them inside the zip.

Section 3 · CodeDeploy

CodeDeploy — Automated Deployment to EC2

  • ▸Deployment strategies — AllAtOnce (fast, has downtime), HalfAtATime (rolling), OneAtATime (safest), Blue/Green
  • ▸Automatic rollback — configure CodeDeploy to roll back if deployment fails or CloudWatch alarm fires
  • ▸Lifecycle hooks — run scripts at 8 points in the deployment lifecycle
  • ▸Deployment groups — target EC2s by tag (e.g., Name=demo-app-server). Add more servers by tagging them.
  • ▸CodeDeploy agent — a daemon running on EC2 that polls for new deployments and executes them
  • ▸Works with — EC2, on-prem servers, Lambda, ECS

Section 3 · appspec.yml

appspec.yml — Tell CodeDeploy How to Deploy

version: 0.0
os: linux

# Copy artifact files to EC2
files:
  - source: app/
    destination: /var/www/html/

permissions:
  - object: /var/www/html/
    pattern: "**"
    owner: apache
    group: apache
    mode: "755"
    type:
      - file
      - directory

hooks:
  BeforeInstall:
    - location: scripts/before_install.sh
      timeout: 60
      runas: root

  AfterInstall:
    - location: scripts/after_install.sh
      timeout: 60
      runas: root

  ApplicationStart:
    - location: scripts/start_server.sh
      timeout: 60
      runas: root

  ValidateService:
    - location: scripts/validate.sh
      timeout: 30
📁 scripts/before_install.sh
#!/bin/bash
systemctl stop httpd || true
rm -rf /var/www/html/*
echo "Old files cleared"
📁 scripts/start_server.sh
#!/bin/bash
systemctl start httpd
systemctl enable httpd
echo "Apache started"
📁 scripts/validate.sh
#!/bin/bash
curl -f http://localhost/ || exit 1
echo "Health check passed"
Section 3 · CodePipeline

CodePipeline — The Orchestrator

  • ▸Fully managed — no infrastructure to manage. Define stages in the console or as code.
  • ▸Event-driven — CodeCommit push triggers the pipeline via CloudWatch Events. Instant, no polling.
  • ▸Artifact passing — each stage's output (zip in S3) is passed as input to the next stage. Immutable.
  • ▸Manual approval actions — add a human gate between staging and prod. Sends SNS notification.
  • ▸Parallel execution — run multiple actions in one stage simultaneously (e.g., test + security scan in parallel)
  • ▸Pricing — $1/active pipeline/month. If you have 5 pipelines (dev/staging/prod + hotfix + release), that's $5/month.
🔔 Pipeline Notifications (Set This Up)

CodePipeline + CodeStar Notifications + SNS = Slack alerts on failure. Every failed build or deploy sends a message to your channel. No surprises.

📋 Pipeline Stage Structure (Console View)
  • ■Stage 1 — Source
    Provider: CodeCommit · Branch: main · Trigger: CloudWatch Events
  • ■Stage 2 — Build
    Provider: CodeBuild · Project: demo-app-build
  • ■Stage 3 — Approve (optional)
    Action: Manual approval · SNS notification
  • ■Stage 4 — Deploy
    Provider: CodeDeploy · App: demo-app · Group: demo-app-dg
🔄 Artifact Flow in S3

Each stage writes to: s3://my-artifacts/pipeline-name/stage-name/artifact.zip. You can download any artifact from any run for debugging.

What Happens at Each Stage

  1. Source stage turns blue (0–15 sec) — EventBridge rule fires on the CodeCommit push. CodePipeline pulls the latest code, zips it, stores as SourceArtifact in S3. Stage turns green.
  2. Deploy stage starts (~15 sec) — CodePipeline passes SourceArtifact to CodeDeploy. The agent on EC2 downloads the zip from S3.
  3. BeforeInstall hook runs — before_install.sh: stops Apache, clears /var/www/html/
  4. Install — CodeDeploy copies app/ files to /var/www/html/
  5. AfterInstall hook runs — after_install.sh: sets file permissions for Apache
  6. ApplicationStart hook runs — start_server.sh: starts and enables Apache
  7. Deploy → green (~1 min total) — all hooks passed. Refresh the browser — updated heading is live.

  • ▸Build once, deploy everywhere — the same artifact from CodeBuild deploys to dev, staging, and prod. No rebuilding.
  • ▸Manual approval gate — add an "Approval" action in CodePipeline between staging and prod. It sends an SNS notification. A human reviews staging and clicks Approve.

Section 5 · Secrets

Secrets Management — Never Hardcode

❌ What Not To Do
# buildspec.yml — DO NOT DO THIS
env:
  variables:
    DB_PASSWORD: "mySuperSecretPass123"
    API_KEY: "sk-abc123xyz"

This gets committed to CodeCommit. Anyone with repo access has your prod credentials.

✅ Use SSM Parameter Store
# Store the secret
$ aws ssm put-parameter \
  --name /myapp/prod/db-password \
  --value "mySuperSecretPass123" \
  --type SecureString

# buildspec.yml — pull at build time
env:
  parameter-store:
    DB_PASSWORD: /myapp/prod/db-password
    API_KEY: /myapp/prod/api-key

# Now use $DB_PASSWORD in build commands
🔐 SSM vs Secrets Manager
  • ▸SSM Parameter Store — free tier available. Good for config + non-rotating secrets.
  • ▸Secrets Manager — $0.40/secret/month. Use for DB credentials that need auto-rotation.
Section 5 · Monitoring

Monitoring & Notifications

Pipeline Notifications

  1. Create an SNS Topic — aws sns create-topic --name cicd-alerts
  2. Subscribe Slack/Email — SNS → Lambda → Slack webhook, or SNS → email endpoint directly
  3. Create CodeStar Notification rule — in CodePipeline console → Notify → Create notification rule → select events: Pipeline execution failed, Stage execution failed
  4. Wire to SNS topic — select your cicd-alerts topic
✅ Result

Every failed pipeline run sends an immediate alert to your Slack channel with the pipeline name, stage that failed, and a direct link to the execution.

CloudWatch Alarms → Auto Rollback

  • ▸Create a CloudWatch alarm on EC2 error rate or 5xx responses from ALB
  • ▸In CodeDeploy deployment group settings: Rollbacks → "Roll back when a deployment fails" AND "Roll back when alarm thresholds are met"
  • ▸Wire the CloudWatch alarm to CodeDeploy
  • ▸If your deployment causes an error rate spike, CodeDeploy automatically redeploys the previous version
⚡ This Is Why CD Beats Manual Deploys

A manual deploy has no automatic rollback. With this setup, a bad deploy that causes errors rolls itself back within minutes — no human needed.

Section 5 · Troubleshooting

Most Common Failures (and How to Fix Them)

ErrorRoot CauseFix
CodeBuild: Access denied to S3 CodeBuild role missing s3:PutObject on artifact bucket Add S3 permission scoped to artifact bucket ARN to CodeBuild role
CodeDeploy: No instances found EC2 tag doesn't match deployment group tag filter Verify EC2 tag Name=demo-app-server matches exactly. Case-sensitive.
CodeDeploy: agent not running codedeploy-agent service stopped or not installed sudo systemctl start codedeploy-agent on EC2. Check /var/log/aws/codedeploy-agent/codedeploy-agent.log
Lifecycle hook timeout Script takes longer than timeout value in appspec.yml Increase timeout. Check if httpd is slow to start. Add logging to hook scripts.
appspec.yml not found appspec.yml not included in CodeBuild artifacts block Add - appspec.yml and - scripts/**/* to artifacts.files in buildspec.yml
Pipeline doesn't trigger on push EventBridge rule is disabled or branch name mismatch Check EventBridge → Rules for the CodePipeline-created rule. Verify branch name is master (not main).
CodeBuild: AccountLimitExceededException — "Cannot have more than 0 builds in queue" Account-level CodeBuild concurrent build quota is 0. Common on new/restricted AWS accounts. Option 1: Open AWS Support → Service limit increase → CodeBuild → Concurrent builds → request 1+. Option 2: Remove Build stage from pipeline (Source → Deploy directly) — valid for static apps that don't need compilation.
Deploy stage fails: "The deployment failed because no instances were found" EC2 instance tag doesn't match deployment group tag filter exactly Go to EC2 → Tags. Verify Name = demo-app-server (case-sensitive). Check CodeDeploy deployment group tag filter matches exactly.
appspec.yml: "The overall deployment failed because too many individual instances failed" One or more lifecycle hook scripts exited with non-zero code Go to CodeDeploy → Deployments → click failed deployment → View events → click the failed lifecycle event → View logs. Fix the script and redeploy.
Section 5 · Production Ready

Making This Pipeline Production Ready

Security & Access

  • ✓IAM least privilege — one scoped role per service. No AdministratorAccess.
  • ✓S3 bucket encryption — enable KMS encryption + block all public access on artifact bucket
  • ✓S3 versioning — enable versioning on artifact bucket. Every build artifact is kept and recoverable.
  • ✓Secrets in SSM / Secrets Manager — never hardcode credentials in buildspec.yml or environment variables
  • ✓CodeBuild in VPC — run builds inside your VPC to access private databases and internal services
  • ✓Branch protection — CodeCommit approval rule: require 1 reviewer before merging to main
  • ✓CloudTrail enabled — every pipeline action, build, and deployment is logged for audit

Reliability & Deployment

  • ✓Multi-environment pipeline — dev → staging → manual approval → prod. Never deploy untested code directly to prod.
  • ✓Blue/Green deployment — CodeDeploy Blue/Green for prod: zero downtime, instant rollback by shifting traffic back
  • ✓Auto-rollback on alarm — wire CloudWatch alarm (error rate / 5xx) to CodeDeploy auto-rollback
  • ✓ValidateService hook — always add a health check script in appspec.yml — deployment fails if the app doesn't respond
  • ✓Pipeline execution mode: SUPERSEDED — new push cancels in-progress run. Prevents stale deploys.
  • ✓Notifications — CodeStar Notifications → SNS → Slack/email on every failure
  • ✓Pipeline as code (Terraform) — define all services as Terraform. Version-control your pipeline definition.
